    时间

    有效时间：5h20m

    • 8:15-8:45，9:00-10:20，1h50m，两道偏简单的算法，本来想做三道的，但想了想，还是尽快把数电搞完吧。
    • 10:55-1:05，2h10m，看 ysyx 状态机相关课。
      • 状态机是看待事物的一种方式。之前内心就一直感觉任何事物都可以分为面向对象与面向过程，例如一本计算机书籍的组成，引导逻辑就像面向过程，重点知识就像面向对象。现在了解了状态机，用一种更加科学的方法刷新了看待事物的方式。

    • 10:30-10:55，25m，状态机部分看书看不懂。
    • 1:55-2:40，45m，大致翻看了下其他课的讲义。（质量似乎很高，而且很有必要，我应该会回来看的。）
    • 4:55-6:35，1h40m，四五章大致看了看，感觉没必要学，先做 PA 看看情况（开天辟地的篇章）。
    • 8:55-9:50，55m，看完PA1.1，实话说，跟没看没啥区别，晚上感觉没什么效率（SB学校，上课耽误时间）

    • 抱怨：耽误我的是学校的课！不然还能多学至少 2h，在课上效率太低了，不如云顶。
    • 理性思考：又不敢旷课，又不舍得花钱找代课，课还是得去的。前端课上学前端是不是性价比高些呢。

    规划

    • 前端得学了，不只是应付考试，前端就是一个很有用的技能。
    • 数据结构得学了，快考试了，至少先把实验题彻底搞懂。
    • 明天再看一遍 PA-RTFSC，似乎得复习 GDB 的使用，然后结合 NEMU 代码导读看源码。前面还有四小时的有用处的课没有看。明天加油！

    微服务的开头讲解的前置知识 MyBatis-Plus，主观感觉封装的太过分了，用魔法的感觉真难受，或许有一部分原因是我这部分的基础不扎实。总觉得还是得了解下 MyBatis 的基本原理，如何封装的 JDBC，而了解原理的最佳方式则是造轮子。从而串联起从 JDBC 到 Mybatis 再到 Mybatis-Plus 大致的底层了解，这样使用起来才会更加得心应手。

    好吧，又查了一下，想看懂 JDBC 似乎还得了解数据库原理，而 CMU 15-445 得花很大力气学。

    用 GPT 查询了下此处知识从计算机底层到编程语言与数据库交互控制的整个知识链，建立系统性的认知：

    • 操作系统有文件管理、网络功能；
    • 编程语言中 C 最接近底层，与操作系统 API 和硬件直接交互，其他语言又加了层抽象；
    • 数据库通过操作系统的文件管理存储数据；
    • TCP/IP 通信，驱动程序通过 TCP，实现应用程序与数据库的交互（SQL）；
    • 编程语言通过 ORM 对象关系映射，实现与数据库表的交互。

    大致就是这样了，GPT 生成的完整内容我存放到了 “OS-APP知识链” 文件中。

    大致了解有意义吗？应该有意义吧，起一个概览全局的作用，然后应该是凹公开课，最重要的一步，最花时间的一步，最难坚持的一步，各种原因，规划不好，导致迟迟没有开始，四级考完就开始吧，争取看完 OS。
