    二、泛型

    概览

    概念

    Java泛型（generics）是JDK5中引入的一个新特性，泛型提供了编译时类型安全监测机制，该机制允许我们在编译时检测到非法的类型数据结构。

    泛型的本质就是参数化类型，也就是所操作的数据类型被指定为一个参数。

    出现原因

    JDK5之前，如果我们没有给集合指定类型，默认认为所有数据类型都是Object类型，此时可以忘记和添加任意的数据类型

    出现两个弊端：

    1. 程序员可以构建一个Object类型的集合，但需要程序员明确知道存储元素的数据类型，然后创建对应的类型变量打印出元素，否则易引发ClassCastException异常，（运行时异常，编译检查不到）。

    2. 在获取数据的时候，无法使用他的特有行为（object类型接收，类比多态不能使用子类的特有行为）

    此时推出了泛型，可以在获取数据的时候就把类型进行统一

    而且我们在获取数据的时候，也省得强转了，非常的方便

    优点

    1. 统一数据类型
    2. 把运行时期的问题提前到了编译期间，避免了强制类型转换可能出现的异常，因为在编译阶段就能确定下来

    拓展

    Java中的泛型是伪泛型

    创建泛型：在集合入口处限制只有String类型能够进入，进入后以Object类型存储

    调用时：再将元素强转为String类型（以String举例）

    泛型信息只存在于代码编译阶段，在进入JVM之前，与泛型相关的信息会被擦除掉，这一过程我们称之为==类型擦除==

    原因：JDK5之前的太麻烦，但还必须要兼容，所以就设置了一个入口来处理

    细节

    • 泛型中不能写基本数据类型
    • 指定泛型的具体类型后，传递数据时，可以传入该类类型或者其子类类型（就是后面的数据具有继承性）
    • 不写泛型的话，类型默认Object类型
    • 泛型类型在逻辑上可以看成是多个不同的类型，但实际上都是相同类型

    定义

    泛型 定义泛型位置 何时确定类型
    泛型类 类名后 使用该类创建对象的时候
    泛型方法 修饰符后 调用该方法的时候
    泛型接口 接口名后 实现类确定类型 或 实现类延续泛型

    成员方法和泛型方法的区别

    成员方法必须强制按照你指定的类型来，用于完成特定的任务

    泛型方法可以使用多个类型参数，它的类型是在调用方法的时候指明的（最主要的区别），适用于多种不同的数据类型，提高了代码的重用性和灵活性

    注意事项

    1. 成员方法不能定义为静态方法 static（因为类中的静态方法还未指明类型）

    泛型方法可以定义为静态方法，泛型方法有返回值，返回值由形参决定，可以定义为static

    总结

    （相当于最重要的两点）

    1. 泛型方法能使方法独立于类而产生变化（所以能用泛型方法就不要用泛型类了）
    2. 如果static方法要使用泛型能力，就必须使其成为泛型方法

    通配符

    什么是类型通配符

    • 类型通配符一般是使用" ? "代替具体的类型实参
    • 所以，类型通配符是类型实参，而不是类型形参

    关键点

    可以限定类型的范围

    应用场景

    在定义类，方法，接口的时候，如果类型不确定，但是能知道以后只能传递某个继承体系重点，就可以使用泛型的通配符

    类型通配符的上限

    • 语法：类/接口<? extends 实参类型>

    要求该泛型的类型，只能是实参类型，或实参类型的子类类型

    • 采用这种形式的集合不能添加元素

    • addAll方法

    类型通配符的下限

    • 语法：类/接口<? super 实参类型>

    要求该泛型的类型，只能是实参类型，或实参类型的父类类型

    • 可以添加元素（包括子类类型）（其实还是不太清楚）

    • 遍历这种形式的集合元素时，都是用Object类型来接收的

    六、Lambda表达式

    语法格式一：无参数，无返回值
    () -> System.out.println("Hello Lambda");

语法格式二：有一个参数，并且无返回值
    (x) -> System.out.println(x);

语法格式三：若只有一个参数，小括号可以省略不写
    x -> System.out.println(x);

语法格式四：有两个以上的参数，有返回值，并且Lambda体中有多条语句
    Compatator<Integer> com = (x,y) -> {
        System.out.println("函数式接口");
        return Integer.compare(x,y);
    };

语法格式五：若Lambda体中只有一条语句，return和大括号都可以省略不写
    Comparator<Integer> com = (x,y) -> Integer.compare(x,y);

语法格式六：Lambda表达式的参数列表的数据类型可以省略不写，因为JVM编译器可以通过上下文推断出数据类型，即“类型推断”
    Comparator<Integer> com = (Integer x,Integer y) -> Integer.compare(x,y);

二、Lambda表达式需要“函数时接口”的支持
重点
函数式接口：接口中只有一个抽象方法的接口，称为函数式接口。
          可以使用注解@FunctionalInterface修饰，来检查是否时函数式接口

    抽象方法只能存在于抽象类或接口中

    在Java中，接口（Interface）和抽象方法（Abstract Method）是面向对象编程的两个重要概念，它们之间存在密切的关系。下面我们来详细讨论一下它们之间的关系：

1. 抽象方法：

    抽象方法是一个在抽象类或接口中声明但没有实现的方法。
    它使用abstract关键字进行声明，没有方法体（即没有花括号和其中的代码）。
    抽象方法只能存在于抽象类或接口中。
    子类继承抽象类后，必须为其中的所有抽象方法提供具体的实现，除非子类本身也是抽象的。

2. 接口：

    接口是一种引用类型，它是方法的集合，这些方法都是抽象的。
    接口使用interface关键字进行声明。
    接口中所有的方法默认都是抽象的，所以不需要使用abstract关键字。
    接口不能包含实例字段或构造函数，但可以包含常量（使用static final修饰的字段）和默认方法（使用default关键字）、静态方法（使用static关键字）以及私有方法（使用private关键字，从Java 9开始支持）。
    一个类可以实现一个或多个接口，通过implements关键字来实现。实现接口的类必须提供接口中所有方法的具体实现。

3. 接口与抽象方法的关系：

    由于接口中的所有方法默认都是抽象的，因此接口和抽象方法之间存在直接的联系。实际上，接口可以被视为一种特殊的抽象类，它只包含抽象方法。
    接口提供了一种机制，使得不同的类可以实现相同的方法集，从而实现多态性。这是面向对象编程的一个重要特性。
    抽象类与接口的一个主要区别在于，抽象类可以包含非抽象方法、实例字段和构造函数，而接口则不能。因此，抽象类提供了更多的灵活性和功能，而接口则更专注于方法的定义和实现的分离。

    总结来说，接口和抽象方法在Java中都是用于实现抽象和定义行为的重要工具。接口是一种特殊的抽象类，它只包含抽象方法。通过实现接口或继承抽象类，子类或实现类可以提供这些抽象方法的具体实现，从而实现多态性和代码的重用。

    八、Stream流

    1. Stream流的作用

    Stream流结合了Lambda表达式，简化集合、数组的操作

    2. Stream流的使用步骤

    1. 先得到一条Stream流，把数据放上去
    获取方式 方法名 说明
    单列集合 default Stream stream() 使用默认方法stream()生成流， default Stream stream()
    双列集合 无 无法直接使用stream流，把Map转成Set集合，间接的生成流（要通过keySet()或entrySet()先转换为单列集合，再转换为stream流）

        //第二个类型：要转成之后的类型

        //apply的形参s：依次表示流里面的每一个数据
        //返回值：表示转换之后的数据

        //当map方法执行完毕之后，流上的数据就变成了整数
        //所以在下面forEach当中，s依次表示流里面的每一个数据，这个数据现在就是整数了
        list.stream().map(new Function<String, Integer>() {
            @Override
            public Integer apply(String s) {
                String[] arr = s.split("-");
                String ageString = arr[1];
                int age = Integer.parseInt(ageString);
                return age;
            }
        }).forEach(s-> System.out.println(s));

    Stream流的终结方法

    名称 说明
    void forEach(Consumer action) 遍历
    long count() 统计
    toArray() 收集流中的数据，放到数组中（两个，一个空参，一个带参数，创建指定数据类型的数组）
    collect(Collector collector) 收集流中的数据，放到集合中（list集合，set集合，map集合都要掌握）

    返回值不是Stream对象，就意味着后续不能再使用中间方法了。所以这些是终结方法。

    九、方法引用

    1. 什么是方法引用？

    把已经存在的方法拿过来用，当作函数式接口中抽象方法的方法体

    1. ::是什么符号？

    方法引用符

    1. 方法引用时要注意什么？（4个点）

    2. 需要有函数式接口

    3. 被引用方法必须已经存在
    4. 被引用方法的形参和返回值需要跟抽象方法保持一致
    5. 被引用方法的功能要满足当前的需求

    方法引用的分类

    • 引用静态方法

    • 格式：类名::静态方法

    • 范例：Integer::parseInt

    • 引用成员方法（格式：对象::成员方法）

    注意，静态方法没有this和super（又该复习了）

    1. 引用其他类的成员方法（其他类对象::成员方法）

    2. 引用本类的成员方法（this::成员方法）

      多个点击事件，将多个else if放到不同的方法中，使代码更加独立，方便日后代码的修改和bug

    3. 引用父类的成员方法（super::成员方法）

    4. 引用构造方法

    为什么要引用构造方法？为了创建对象

    练习|应用场景：将数据封装成对象并收集到List集合中

    • 格式：类名::new

    • 范例：Student::new

    • 其他调用方法

    • 使用类名引用成员方法

    • 引用数组的构造方法
